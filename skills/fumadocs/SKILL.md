# Fumadocs — Next.js Documentation Framework

## Package Versions (as of 2026-03)

Use these compatible versions together:

```json
{
  "fumadocs-core": "^15.8.5",
  "fumadocs-mdx": "^14.2.11",
  "fumadocs-ui": "^15.8.5"
}
```

**fumadocs-ui v15** supports Tailwind CSS v4 (`^3.4.14 || ^4.0.0`).
**fumadocs-ui v14** only supports Tailwind CSS v3 — do NOT use with Tailwind v4 projects.

## Setup Checklist

### 1. Install packages

```bash
npm install fumadocs-core fumadocs-ui fumadocs-mdx
```

### 2. `source.config.ts` (project root)

```ts
import { defineDocs, defineConfig } from 'fumadocs-mdx/config';

export const { docs, meta } = defineDocs({
  dir: 'content/docs',
});

export default defineConfig();
```

### 3. `next.config.ts` — wrap with createMDX

```ts
import type { NextConfig } from "next";
import { createMDX } from "fumadocs-mdx/next";

const nextConfig: NextConfig = {
  // your config
};

const withMDX = createMDX();
export default withMDX(nextConfig);
```

### 4. `src/lib/docs/source.ts` — use toFumadocsSource (NOT createMDXSource)

```ts
import { docs, meta } from '@/.source';
import { toFumadocsSource } from 'fumadocs-mdx/runtime/server';
import { loader } from 'fumadocs-core/source';

export const source = loader({
  baseUrl: '/docs',
  source: toFumadocsSource(docs, meta),
});
```

⚠️ `createMDXSource` was removed in fumadocs-mdx v14. Use `toFumadocsSource` instead.

### 5. `tsconfig.json` — path alias + include

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"],
      "@/.source": ["./.source/server"]
    }
  },
  "include": [
    "next-env.d.ts",
    "**/*.ts",
    "**/*.tsx",
    ".next/types/**/*.ts",
    ".source/**/*.ts"
  ]
}
```

### 6. `package.json` — generate step (CRITICAL)

The `.source/` directory is generated by fumadocs-mdx and must exist before the build:

```json
{
  "scripts": {
    "build": "fumadocs-mdx && next build",
    "postinstall": "fumadocs-mdx"
  }
}
```

Without this, builds fail with: `Module not found: Can't resolve '@/.source'`

### 7. `.gitignore`

```
.source/
```

The `.source/` directory is generated — never commit it.

### 8. CSS imports (`globals.css`)

**fumadocs-ui v15:**
```css
@import "tailwindcss";
@import "fumadocs-ui/style.css";
```

**fumadocs-ui v14 (legacy):**
```css
@import "tailwindcss";
@import "fumadocs-ui/dist/style.css";
```

⚠️ `fumadocs-ui/css/neutral.css` and `fumadocs-ui/css/preset.css` do NOT exist. These paths cause build failures.

## Layout — v15 API

```tsx
import { DocsLayout } from 'fumadocs-ui/layouts/docs';

// ✅ Correct: links at top level
<DocsLayout
  tree={source.pageTree}
  nav={{ title: <span>My Docs</span>, url: '/docs' }}
  links={[
    { text: 'Home', url: '/', external: false },
  ]}
  sidebar={{ defaultOpenLevel: 1 }}
>
  {children}
</DocsLayout>

// ❌ Wrong: links inside nav (v14 API, removed in v15)
<DocsLayout nav={{ links: [...] }} />
```

## Page Component

```tsx
import { source } from '@/lib/docs/source';
import { DocsPage, DocsBody, DocsTitle, DocsDescription } from 'fumadocs-ui/page';
import defaultMdxComponents from 'fumadocs-ui/mdx';
import { notFound } from 'next/navigation';

export default async function Page(props: { params: Promise<{ slug?: string[] }> }) {
  const params = await props.params;
  const page = source.getPage(params.slug);
  if (!page) notFound();

  const MDX = page.data.body;

  return (
    <DocsPage toc={page.data.toc}>
      <DocsTitle>{page.data.title}</DocsTitle>
      <DocsDescription>{page.data.description}</DocsDescription>
      <DocsBody>
        <MDX components={{ ...defaultMdxComponents }} />
      </DocsBody>
    </DocsPage>
  );
}

export function generateStaticParams() {
  return source.generateParams();
}
```

## Search

In v15, `createFromSource` requires a `buildIndex` function to access `structuredData`:

```ts
// src/app/api/search/route.ts
import { source } from '@/lib/docs/source';
import { createFromSource } from 'fumadocs-core/search/server';

export const { GET } = createFromSource(source, {
  buildIndex: (page) => {
    const data = page.data as any;
    return {
      id: page.url,
      title: data.title ?? '',
      description: data.description,
      url: page.url,
      structuredData: data.structuredData,
    };
  },
});
```

⚠️ Without `buildIndex`, you get: `Cannot find structured data from page, please define the page to index function`

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `Can't resolve '@/.source'` | `.source/` not generated | Add `fumadocs-mdx` to build + postinstall scripts |
| `peerOptional tailwindcss@"^3.4.14"` | fumadocs-ui v14 with Tailwind v4 | Upgrade to fumadocs-ui v15 |
| `'createMDXSource' is not exported` | Removed in fumadocs-mdx v14 | Use `toFumadocsSource` from `fumadocs-mdx/runtime/server` |
| `'links' does not exist in type NavOptions` | v14→v15 API change | Move `links` from `nav` prop to top-level `DocsLayout` prop |
| `fumadocs-ui/css/neutral.css` not found | Wrong import path | Use `fumadocs-ui/style.css` (v15) or `fumadocs-ui/dist/style.css` (v14) |
| `Cannot find structured data from page` | Search needs buildIndex in v15 | Add `buildIndex` to `createFromSource` (see Search section) |

## Content Structure

```
content/
  docs/
    meta.json          ← sidebar navigation
    index.mdx          ← /docs landing page
    guides/
      quickstart.mdx   ← /docs/guides/quickstart
    api/
      fleet.mdx        ← /docs/api/fleet
```

### meta.json (sidebar order)

```json
{
  "root": true,
  "pages": [
    "---Getting Started---",
    "index",
    "guides/quickstart",
    "---API Reference---",
    "api/..."
  ]
}
```
